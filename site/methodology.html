METHOD / STANDARD

方法与标准

OpenODC 衡量的是运行边界被公开资料说清楚了多少，不是给智驾能力排名，也不是替代厂家官方安全声明。

阅读样例前需要知道

除非记录标记为 vendor_confirmed，OpenODC 样例均为社区基于公开资料提取。L2 样例描述功能适用条件，不能解释为车辆可脱离驾驶员监管；L3/L4 样例才涉及 ADS 在 ODD 内承担动态驾驶任务。

OpenODC 做什么

公开资料结构化

把车主手册、官方页面、App 规则、政府公告、媒体测评中的运行边界，映射到 GB/T 45312—2025 的 144 个 ODC 要素。

缺口度量

明确哪些要素有官方或手册依据，哪些只是社区整理或推定，哪些在公开资料中没有被明确说明。

横向比较

用同一套元素体系比较不同车型、不同功能、不同自动化等级的公开边界差异，避免只比较功能名。

ODC 与 ODD 的关系

ODD 通常描述自动驾驶系统可以运行的道路、环境、交通、速度、地理围栏等设计运行域。GB/T 45312—2025 使用 ODC 元素体系，把这些边界拆成可填写、可校验、可对比的条件清单。OpenODC 将这种清单做成 JSON Schema 和网页工具。

对象OpenODC 中的含义主要风险
L2 辅助驾驶功能建议使用条件和退出/禁用条件；驾驶员持续负责动态驾驶任务把“可用条件”误读成“系统负责驾驶”
L3 条件自动驾驶ADS 在 ODD 内承担动态驾驶任务的前提边界，同时需要定义接管请求和后备用户责任只写道路和天气，不写 fallback 条件
L4 Robotaxi / ADS服务运营边界，通常包括 geofence、时段、天气、远程协助、监管许可和 App 实时规则把示范运营区域误读成全国通用能力

ODC 表格的局限

ODC 表格是公开透明的必要起点，但它不是完整的系统安全边界。表格中的道路、天气、标线、目标物、速度等要素，本质上是把高维真实场景投影到一个个单项维度。真实道路风险往往来自多个要素同时出现，不能把每个单项允许范围简单相乘，推定为组合场景也一定可用。

层级OpenODC 中的处理边界提醒
单项 ODC 要素仍然是主表：逐项说明允许、不允许、参数范围和证据来源解决“公开资料有没有说清楚”
元素关联关系保留标准中的主从元素关系，例如某道路类型限制速度或设施条件解决“单项之间有没有显式约束”
边界组合新增轻量补充层：列出少量公开资料可支撑的典型组合边界解决“单项允许不等于组合可用”

边界组合与触发条件

OpenODC 不把自己扩展成完整 SOTIF 分析平台，但会把公开资料中已经能识别的典型组合边界标出来。面向消费者，它们叫“边界组合”；面向开发者，它们可以作为 Trigger Condition candidates，也就是触发条件候选。

不穷尽场景空间

每个样例只列少量高价值组合，例如雨雪雾 + 标线不清、施工区 + 临时车道、geofence + App 实时规则。

不替代厂家安全分析

边界组合来自公开资料和社区整理，只用于提示组合边界、公开缺口和验证入口，不构成 SOTIF 安全结论。

连接验证与沟通

消费者看到“什么时候更该谨慎”，开发者看到“哪些组合适合转成测试场景或触发条件分析”。

与 ROAM / DRIVEResearch 的关系

如果只看 ODC 表格，容易停留在“某个单项要素有没有写清楚”。但真实道路中的触发条件通常是组合出来的：同样是雨天，和夜间、逆光、车道线磨损、施工改道、驾驶员分心叠加后，风险含义完全不同。因此 OpenODC 只保留轻量的组合边界层，把更重的暴露评估和异常结果分析交给互补项目承接。

问题层对应项目边界组合承担的作用
厂家或公开资料声明了什么边界OpenODC把单项 ODC 要素连接成可引用的组合边界候选，明确证据来源和公开缺口
这些组合在真实道路中有多常见DRIVEResearch用自然驾驶数据估计组合出现频次、参数分布和人类驾驶基准
越界或接近边界后发生了什么ROAM用远程运营异常、事故和干预记录反查哪些组合值得优先治理

证据等级

标签含义是否计入公开资料覆盖
官方声明厂家官网、官方新闻稿、官方配置表、Robotaxi 官方运营规则计入，置信度最高
手册明确车主手册或使用说明书直接写明的条件、阈值、警示计入，置信度最高
社区整理公开资料能支撑方向性判断，但未给出完整量化阈值计入，但必须可追溯来源
推定基于系统架构、法规、行业常识或公开材料间接推导计入覆盖，但必须单独显示，不能当作官方披露
公开资料未明确公开来源没有直接说明该要素不计入覆盖，是项目要揭示的缺口
结构性国标层级中的父类节点，具体判断看下级元素不计入实质覆盖

覆盖度算法

Gallery 卡片中的覆盖度不是厂家披露率。它表示 OpenODC 基于公开资料能为多少个国标要素建立解释：官方声明 + 手册明确 + 社区整理 + 推定。页面会同时显示公开资料未明确的数量，避免把缺口误认为能力。
