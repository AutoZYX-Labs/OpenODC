VENDOR INTAKE WORKBENCH

Vendor ODC Intake Workbench

The practical workflow is not asking engineers to fill 144 elements one by one. Import manuals, configuration tables, operating rules, or sanitized excerpts first; generate an ODC draft table; then let authorized vendor contacts confirm and publish.

Public-site boundary This page does not perform real vendor authentication or upload sensitive files. It only processes public or sanitized text in the browser. Official vendor submission should use domain verification, role approval, and back-office review; GitHub remains only a community fallback path.
01

Organization Verification

Verify submitters through corporate email domains, vendor-admin invitations, or trusted partner allowlists. Selecting a vendor name on a public page is not authentication.

02

Source Import

Import owner manuals, feature guides, configuration tables, robotaxi operating rules, OTA notes, or sanitized internal boundary documents.

03

ODC Extraction

Map text to the 144 elements in GB/T 45312—2025, including evidence snippets, parameter ranges, and unspecified gaps.

04

Confirm and Publish

After vendor confirmation, publish through a portal, API, CSV review package, or GitHub community contribution path.

MANUAL TO ODC TABLE

Generate an ODC Draft Table

Browser-local processing

Expected Output

  • Status for every standard element: permitted, not permitted, unspecified, or structural
  • Evidence snippets and source URLs, not just conclusions
  • Quantified ranges for speed, rainfall, visibility, road type, and other key boundaries
  • Review actions for vendor confirmation, legal sanitization, or missing manual page references

This is closer to how engineering teams work: the system performs the first extraction pass, and humans confirm, correct, and add evidence.

Publication Should Not Depend on GitHub Alone

Target Mode

Vendor Portal

Corporate-domain verification, admin approval, role permissions, version freeze, legal sanitization, and publication confirmation happen inside the portal.

Near-Term

Review Package

Export CSV / JSON / Markdown and submit it through email, enterprise IM, or a web form for OpenODC administrator review.

At Scale

API / Batch Sync

Suitable for OEMs and Tier 1 suppliers maintaining many models, markets, feature versions, SOP releases, and OTA updates.

Community Path

GitHub PR

Useful for open-source contributors and researchers who already use GitHub; it should not be the only path for vendor engineers.
