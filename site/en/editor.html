② Editing Steps
  1. Select elements from the taxonomy tree, then set each item to permitted or not permitted.
  2. Not-permitted items should specify exit behavior where applicable.
  3. For permitted items, add quantitative ranges when public evidence supports them; otherwise mark public evidence gaps explicitly.
  4. L2 records describe feature use conditions only; they must not imply that the system performs the dynamic driving task without the driver.
  5. The JSON preview updates live. Export JSON or Markdown when ready.
  6. Submit the JSON file under data/examples/ through the GitHub repository.
